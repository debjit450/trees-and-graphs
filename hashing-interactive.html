Concepts

Direct Access

Hashing maps data of arbitrary size to fixed-size values (indices).

Goal: O(1) Average Search Time.

Comprehensive Guide

The Magic of O(1) Lookups.

Hash Functions

A hash function takes an input (or 'key') and returns a fixed-size string of bytes, typically a digest that is unique to each unique input. In data structures, we map this digest to an integer index.

Key Components

  • Hash Function: Converts Key → Integer Index.
  • Bucket Array: The array where data is stored.
  • Load Factor: (Number of Entries) / (Array Size). Impact performance.

Good Properties

A good hash function should uniformly distribute keys across the bucket array to minimize collisions (e.g., key % prime_number).

Collision Resolution

Since the bucket array is finite, two different keys may hash to the same index (Pigeonhole Principle). We must handle this.

1. Separate Chaining

Each bucket stores a linked list (or dynamic array) of entries. If collision occurs, append to the list.

Pros: Simple, table never "fills up".
Cons: Cache performance (pointers).

2. Open Addressing

All elements are stored in the hash table itself. If collision occurs, probe for next empty slot.

  • Linear Probing: Check i+1, i+2...
  • Quadratic Probing: Check i+1², i+2²...

Complexity Analysis

Average Search O(1)
Average Insert O(1)
Worst Case O(N)
Worst case happens if all keys hash to the same index.
