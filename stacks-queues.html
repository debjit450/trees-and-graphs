Understanding Linear Data Structures: Stacks, Queues, and Deques.

The Stack

A Stack is a linear data structure that follows the LIFO (Last In, First Out) principle. Think of it like a stack of plates in a cafeteria: you can only take the top plate, and you place new plates on top.

Key Operations

  • Push(x): Adds an element x to the top of the stack.
  • Pop(): Remover the top element.
  • Peek() / Top(): Returns the top element without removing it.
  • isEmpty(): Checks if the stack is empty.

Real-World Applications

  • Function Call Stack: Tracking active subroutines in programming languages (Recursion).
  • Undo Mechanism: Text editors store history in a stack.
  • Syntax Parsing: Checking balanced parentheses in code ((a+b)*c).

Implementation (Array-based)

class Stack { int top = -1; int[] arr = new int[100]; push(int x) { arr[++top] = x; } pop() { if (top == -1) throw new Error("Underflow"); return arr[top--]; } }

The Queue

A Queue follows the FIFO (First In, First Out) principle. It behaves like a line at a ticket counter: people join at the back (Rear) and leave from the front (Front).

Key Operations

  • Enqueue(x): Adds element to the Rear.
  • Dequeue(): Removes element from the Front.
  • Front(): Peeks at the first element.

Implementation Nuance

Using a simple Array for a Queue is inefficient for Dequeue (requires shifting elements O(N)).

Solution: Use a Circular Queue or Linked List to achieve O(1) for both operations.

Applications

  • CPU Scheduling: Processes waiting for CPU time.
  • Printer Spooling: Documents are printed in the order received.
  • BFS Algorithm: Breadth-First Search in Graphs.

Monotonic Stack

A specialized use of a stack where elements are maintained in a specific order (increasing or decreasing). This technique is powerful for solving "Next Greater Element" type problems in linear time O(N).

The Logic

To find the next greater element for every item:

  1. Iterate through the array.
  2. While the stack is not empty AND current element > stack top element:
    • The current element IS the "Next Greater" for the stack top.
    • Pop the stack.
  3. Push the current index onto the stack.
Classic Problem

Daily Temperatures: Given a list of daily temperatures, return a list such that, for each day, you identify how many days you would have to wait until a warmer temperature.

Solved efficiently using a Monotonic Stack.
