Discovery & Low Values

Node disc[] low[]

Definitions

Articulation Point

A vertex whose removal disconnects the graph (increases number of components).

Bridge

An edge whose removal disconnects the graph.

These are critical points in network design - they represent single points of failure!

Tarjan's Algorithm

Key Values

  • disc[u]: Discovery time of node u
  • low[u]: Lowest disc reachable from subtree of u
function tarjan(u, parent): disc[u] = low[u] = time++ children = 0 for v in adj[u]: if disc[v] == -1: // Unvisited children++ tarjan(v, u) low[u] = min(low[u], low[v]) // Check articulation point if parent == null && children > 1: u is articulation point if parent != null && low[v] >= disc[u]: u is articulation point // Check bridge if low[v] > disc[u]: (u, v) is bridge else if v != parent: low[u] = min(low[u], disc[v])

Time: O(V + E)

Key Insights

  • Root is AP if it has ≥2 children in DFS tree
  • Non-root u is AP if child v has low[v] ≥ disc[u]
  • (u,v) is bridge if low[v] > disc[u]
  • Back edges help lower the low[] values

Applications

  • Network reliability - finding weak points
  • Transportation - critical roads/intersections
  • Biconnected components
  • Server redundancy planning
