What are Articulation Points and Bridges?

In an undirected connected graph:

Articulation Point (Cut Vertex)

A vertex whose removal disconnects the graph. Removing it increases the number of connected components.

Bridge (Cut Edge)

An edge whose removal disconnects the graph. It's the only path between some pair of vertices. 

Example Graph:
                     
    1 ─── 2 ─── 3 ─── 4
          │           │
          5 ────────── 6
                     
Articulation Points: {2, 3}
  - Removing 2 disconnects {1} from rest
  - Removing 3 disconnects {4,6} from rest
  
Bridges: {(1,2), (2,3), (3,4), (3,6)} 
  - Wait, let's check: 3-6 exists but 4-6 also exists
  
Actually:
    1 ─── 2 ─── 3 ─── 4
          │           │
          5           6
          
Bridges: {(1,2), (2,5), (2,3), (3,4), (3,6)}
All edges are bridges in a tree!

Why Are They Important?

Network Reliability

Critical points in computer/communication networks. Failure = network split.

Infrastructure Planning

Critical roads/bridges in transportation networks.

Attack Vulnerability

Identifying weak points that adversaries might target.

Graph Decomposition

Finding biconnected components for graph analysis.

Key Concepts

Discovery Time (disc)

The timestamp when a vertex is first visited during DFS.

Low Value (low)

The minimum discovery time reachable from the subtree rooted at this vertex, including via back edges.

Low Value Definition

low[v] = min(disc[v], disc[ancestor reachable via back edge], low[children])

Intuitively: the "earliest" vertex we can reach from v's subtree through back edges.

Back Edge

An edge to an already-visited ancestor in the DFS tree (creates a cycle).

Finding Articulation Points

Conditions for Articulation Point

A vertex u is an articulation point if:

  1. u is root of DFS tree AND has 2+ children
  2. u is not root AND has a child v where low[v] ≥ disc[u]
    • This means v's subtree cannot reach any ancestor of u
    • Removing u disconnects v's subtree from the rest
time = 0 disc = [-1] * n low = [-1] * n parent = [-1] * n articulationPoints = set() function findArticulationPoints(graph, n): for v = 0 to n-1: if disc[v] == -1: dfs(v, graph) return articulationPoints function dfs(u, graph): global time children = 0 disc[u] = low[u] = time time++ for v in graph[u]: if disc[v] == -1: // Not visited (tree edge) children++ parent[v] = u dfs(v, graph) // Update low value from child low[u] = min(low[u], low[v]) // Check articulation point conditions if parent[u] == -1 and children > 1: // Root with 2+ children articulationPoints.add(u) if parent[u] != -1 and low[v] >= disc[u]: // Non-root with cut condition articulationPoints.add(u) else if v != parent[u]: // Back edge (not to parent) low[u] = min(low[u], disc[v])

Finding Bridges

Condition for Bridge

An edge (u, v) is a bridge if: low[v] > disc[u]

This means v's subtree has NO back edge to u or any ancestor of u. The edge (u,v) is the only connection.

Difference from Articulation Points

Bridge: low[v] > disc[u] (strictly greater)

Articulation Point: low[v] ≥ disc[u] (greater or equal)

If low[v] = disc[u], there's a back edge to u itself — edge (u,v) is not a bridge, but u might be an articulation point.

time = 0 disc = [-1] * n low = [-1] * n parent = [-1] * n bridges = [] function findBridges(graph, n): for v = 0 to n-1: if disc[v] == -1: dfs(v, graph) return bridges function dfs(u, graph): global time disc[u] = low[u] = time time++ for v in graph[u]: if disc[v] == -1: // Tree edge parent[v] = u dfs(v, graph) low[u] = min(low[u], low[v]) // Bridge condition: strictly greater if low[v] > disc[u]: bridges.append((u, v)) else if v != parent[u]: // Back edge low[u] = min(low[u], disc[v])

Biconnected Components

A biconnected component is a maximal subgraph with no articulation points. It cannot be disconnected by removing any single vertex.

Properties

  • A graph is biconnected if it has no articulation points
  • Bridges form their own biconnected components
  • Articulation points can belong to multiple biconnected components
Graph:
    1 ─── 2 ─── 3
    │     │     
    └──── 4 ─── 5
    
Biconnected Components:
BC1: {1, 2, 4} (cycle)
BC2: {2, 3} (edge, 2 is articulation point)
BC3: {4, 5} (edge, 4 is articulation point)

Finding Biconnected Components

Use a stack during DFS to collect edges. When an articulation point is found, pop edges to form a component.

Handling Multiple Edges

Multiple Edges Between Same Vertices

If there are multiple edges between u and v, the edge (u,v) is NOT a bridge — removing one edge still leaves another. Track edge indices instead of just checking if v ≠ parent[u].

For multigraphs, instead of v != parent[u], track the edge ID and ensure we don't go back on the same edge we came from:

// For multigraphs: track edge ID function dfs(u, parentEdge, graph): disc[u] = low[u] = time++ for (v, edgeId) in graph[u]: if disc[v] == -1: dfs(v, edgeId, graph) low[u] = min(low[u], low[v]) if low[v] > disc[u]: bridges.add(edgeId) else if edgeId != parentEdge: // Different edge low[u] = min(low[u], disc[v])
