What is Topological Sorting?

A topological sort (or topological ordering) of a directed acyclic graph (DAG) is a linear ordering of its vertices such that for every directed edge (u → v), vertex u comes before v in the ordering.

Key Insight

Topological sorting is only possible for DAGs (Directed Acyclic Graphs). If the graph has a cycle, no valid topological ordering exists because there's no way to order vertices in a cycle linearly.

Example DAG:          Topological Orders:
                      
    A → B → D         A, B, C, D, E  ✓
    ↓   ↓             A, C, B, D, E  ✓
    C → E             A, C, B, E, D  ✓
                      (Multiple valid orderings possible)

Real-World Applications

Build Systems

Determining compilation order where files depend on each other (Make, Maven, Gradle).

Task Scheduling

Ordering tasks with prerequisites (course prerequisites, project management).

Package Management

Installing dependencies in correct order (npm, pip, apt).

Spreadsheet Formulas

Evaluating cells that reference other cells in correct order.

Algorithm 1: Kahn's Algorithm (BFS-based)

Kahn's algorithm uses the concept of in-degree (number of incoming edges) to find vertices with no dependencies.

How It Works

  1. Calculate in-degree for all vertices
  2. Add all vertices with in-degree 0 to a queue (no dependencies)
  3. While queue is not empty:
    • Remove vertex u from queue, add to result
    • For each neighbor v of u, decrement v's in-degree
    • If v's in-degree becomes 0, add v to queue
  4. If result contains all vertices → valid topological order. Otherwise → cycle exists!
function kahnTopologicalSort(graph, n): inDegree = array of size n, initialized to 0 // Calculate in-degrees for u = 0 to n-1: for v in graph[u]: inDegree[v]++ // Add all vertices with in-degree 0 queue = new Queue() for v = 0 to n-1: if inDegree[v] == 0: queue.enqueue(v) result = [] while queue not empty: u = queue.dequeue() result.append(u) for v in graph[u]: inDegree[v]-- if inDegree[v] == 0: queue.enqueue(v) // Check if all vertices processed if len(result) != n: return null // Cycle detected! return result

Cycle Detection

If the result doesn't contain all n vertices after Kahn's algorithm completes, the graph contains a cycle. Vertices in the cycle can never have in-degree 0.

Algorithm 2: DFS-based Approach

The DFS-based approach uses the observation that in a topological order, a vertex appears before all vertices reachable from it. We add vertices to the result in reverse order of their DFS finish times.

How It Works

  1. Perform DFS from each unvisited vertex
  2. When a vertex finishes (all descendants processed), push it to a stack
  3. Pop all vertices from stack to get topological order
function dfsTopologicalSort(graph, n): visited = array of size n, initialized to false inStack = array of size n, initialized to false // For cycle detection stack = [] for v = 0 to n-1: if not visited[v]: if not dfs(v, graph, visited, inStack, stack): return null // Cycle detected // Reverse stack to get topological order return reverse(stack) function dfs(u, graph, visited, inStack, stack): visited[u] = true inStack[u] = true // Currently in recursion stack for v in graph[u]: if not visited[v]: if not dfs(v, graph, visited, inStack, stack): return false // Cycle found else if inStack[v]: return false // Back edge = cycle! inStack[u] = false // Done with this path stack.push(u) // Add to result after processing children return true

Graph:  5 → 0 ← 4
        ↓       ↓
        2 → 3 → 1

Edges: 5→0, 4→0, 4→1, 2→3, 3→1, 5→2

Initial in-degrees: [2, 2, 1, 1, 0, 0]
                     0  1  2  3  4  5

Init [4, 5] - - []
1 [5] 4 inDeg[0]=1, inDeg[1]=1 [4]
2 [0] 5 inDeg[0]=0, inDeg[2]=0 [4, 5]

Aspect Kahn's Algorithm (BFS) DFS-based

Counting Topological Orders

A DAG may have multiple valid topological orderings. The number depends on the graph structure:

  • Linear chain: Only 1 ordering
  • No edges: n! orderings (any permutation)
  • General DAG: Count using DP over subsets (exponential)

Getting Lexicographically Smallest Order

In Kahn's algorithm, replace the regular queue with a min-priority queue (min-heap). This ensures we always process the smallest-numbered vertex first among those with in-degree 0.

Detecting Cycles

Both algorithms can detect cycles in directed graphs:

Using Kahn's Algorithm

If the final result has fewer than V vertices, there's a cycle (some vertices never reached in-degree 0).

Using DFS

Use three colors for vertices:

  • White: Unvisited
  • Gray: Currently in recursion stack
  • Black: Completely processed

A back edge (edge to a gray vertex) indicates a cycle!
