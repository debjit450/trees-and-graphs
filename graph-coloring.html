What is Graph Coloring?

Graph coloring is the assignment of labels (called "colors") to vertices of a graph such that no two adjacent vertices share the same color. This is known as vertex coloring. 

Valid 3-coloring:          Invalid coloring:
                           
   (R)───(B)                  (R)───(R)  ✗
   / \   / \                  / \   / \
 (B) (G)─(R)                (B) (G)─(R)
                           
R=Red, B=Blue, G=Green     Adjacent vertices same color!

Key Insight

The minimum number of colors needed to color a graph is called its chromatic number, denoted χ(G). Finding the chromatic number is NP-hard for general graphs.

Key Concepts

Chromatic Number χ(G)

The minimum number of colors needed to properly color a graph.

k-Colorable

A graph is k-colorable if it can be properly colored using at most k colors.

Bipartite = 2-Colorable

A graph is bipartite if and only if it's 2-colorable (no odd-length cycles).

Complete Graph Kn

Requires exactly n colors (every vertex connected to every other).

Bipartite Graph Check (2-Coloring)

A graph is bipartite if we can partition its vertices into two sets such that every edge connects a vertex in one set to a vertex in the other. This is equivalent to being 2-colorable.

BFS-based Bipartite Check

function isBipartite(graph, n): color = [-1] * n // -1 = uncolored, 0 or 1 = color for start = 0 to n-1: if color[start] == -1: // BFS from this uncolored vertex queue = [start] color[start] = 0 while queue not empty: u = queue.dequeue() for v in graph[u]: if color[v] == -1: // Color with opposite color color[v] = 1 - color[u] queue.enqueue(v) else if color[v] == color[u]: // Same color = not bipartite! return false return true

DFS-based Bipartite Check

function isBipartiteDFS(graph, n): color = [-1] * n for v = 0 to n-1: if color[v] == -1: if not dfs(v, 0, graph, color): return false return true function dfs(u, c, graph, color): color[u] = c for v in graph[u]: if color[v] == -1: if not dfs(v, 1 - c, graph, color): return false else if color[v] == c: return false return true

Bipartite Condition

A graph is bipartite if and only if it contains no odd-length cycles. If we find an edge where both endpoints have the same color during BFS/DFS, we've found an odd cycle.

Greedy Coloring Algorithm

The greedy algorithm colors vertices one by one, assigning each vertex the smallest available color that doesn't conflict with already-colored neighbors.

function greedyColoring(graph, n): color = [-1] * n // -1 = uncolored for u = 0 to n-1: // Find colors used by neighbors usedColors = set() for v in graph[u]: if color[v] != -1: usedColors.add(color[v]) // Find smallest unused color c = 0 while c in usedColors: c++ color[u] = c return color

Properties of Greedy Coloring

  • Time Complexity: O(V + E)
  • Colors Used: At most Δ + 1 (Δ = maximum degree)
  • Not Optimal: May use more colors than chromatic number
  • Order Matters: Different vertex orderings give different results

Not Always Optimal

Greedy coloring doesn't always find the minimum number of colors. The result depends on the order in which vertices are processed. However, it's guaranteed to use at most Δ + 1 colors.

The Four Color Theorem

The Four Color Theorem states that any planar graph (a graph that can be drawn on a plane without edge crossings) can be colored with at most 4 colors.

Map Coloring

This theorem originated from map coloring: any map can be colored with 4 colors such that no two adjacent regions share the same color. This was proven in 1976 using computer assistance — one of the first major theorems proved with computers.

Applications

Scheduling

Exam scheduling where conflicting exams (shared students) need different time slots.

Register Allocation

Assigning CPU registers to variables in compilers.

Map Coloring

Coloring regions on a map so adjacent regions have different colors.

Frequency Assignment

Assigning radio frequencies to transmitters to avoid interference.

Sudoku

Sudoku can be modeled as a graph coloring problem with 9 colors.

Pattern Matching

Graph isomorphism and subgraph matching algorithms.

NP-Completeness

Determining whether a graph can be colored with exactly k colors (for k ≥ 3) is NP-complete. This means no known polynomial-time algorithm exists, and finding one would prove P = NP!

Edge Coloring

Besides vertex coloring, there's also edge coloring: assigning colors to edges such that no two edges sharing a vertex have the same color.

Vizing's Theorem

For any simple graph G with maximum degree Δ, the edges can be colored with either Δ or Δ + 1 colors.

  • Class 1 graphs: Can be edge-colored with Δ colors
  • Class 2 graphs: Require Δ + 1 colors
