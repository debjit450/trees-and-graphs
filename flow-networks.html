What is a Flow Network?

A flow network (or network flow) is a directed graph where each edge has a capacity, and flow can be sent from a source vertex to a sink vertex. The goal is typically to find the maximum amount of flow that can be sent from source to sink.

Flow Network Example:
                     capacity
         ┌──────(10)──────┐
         │                ↓
    [S] ─┼───(15)───→ [A] ───(10)───→ [T]
         │            ↗  │
         │     (5)   /   │(5)
         │      ↓   /    ↓
         └───→ [B] ─────(10)────→ [C]
              (10)
                     
S = Source, T = Sink (Target)
Numbers = Edge Capacities

Key Terminology

Source (s)

The starting vertex where flow originates. Has only outgoing edges in typical formulations.

Sink (t)

The destination vertex where flow ends. Has only incoming edges.

Capacity c(u,v)

Maximum flow that can pass through edge (u,v).

Flow f(u,v)

Actual flow through edge (u,v). Must satisfy: 0 ≤ f(u,v) ≤ c(u,v)

Flow Constraints

A valid flow must satisfy these constraints:

  1. Capacity Constraint: For every edge, 0 ≤ f(u,v) ≤ c(u,v)
  2. Flow Conservation: For every vertex except s and t, incoming flow = outgoing flow
    ∑ f(u,v) = ∑ f(v,w) for all v ∉ {s, t}

Flow Value

The value of a flow |f| is the total flow out of source s (or equivalently, into sink t):
|f| = ∑ f(s,v) - ∑ f(v,s)

Max-Flow Min-Cut Theorem

The Theorem

The maximum value of an s-t flow equals the minimum capacity of an s-t cut.

What is a Cut?

An s-t cut is a partition of vertices into two sets S and T where s ∈ S and t ∈ T. The capacity of a cut is the sum of capacities of edges going from S to T.

Min Cut Example:
                     
    [S] ──(10)──→ [A] ──(5)──┐
     │             │         ↓
    (8)           (3)      [T]
     ↓             ↓         ↑
    [B] ──(15)──→ [C] ──(7)──┘

Min Cut: edges {(A,T), (C,T)} with capacity 5 + 7 = 12
Max Flow = 12

Ford-Fulkerson Method

The Ford-Fulkerson method is a general approach to finding maximum flow by repeatedly finding augmenting paths and pushing flow along them.

Residual Graph

The residual graph Gf shows remaining capacity on each edge:

  • Forward edge: residual capacity = c(u,v) - f(u,v)
  • Backward edge: residual capacity = f(v,u) (allows "undoing" flow)

Augmenting Path

A path from s to t in the residual graph with positive capacity on all edges.

function fordFulkerson(graph, source, sink): maxFlow = 0 residual = copy(graph) // Residual graph while true: // Find augmenting path (DFS/BFS) path = findPath(residual, source, sink) if path == null: break // No more augmenting paths // Find bottleneck (minimum capacity on path) bottleneck = infinity for (u, v) in path: bottleneck = min(bottleneck, residual[u][v]) // Augment flow along path for (u, v) in path: residual[u][v] -= bottleneck // Reduce forward edge residual[v][u] += bottleneck // Increase backward edge maxFlow += bottleneck return maxFlow

Time Complexity: O(E × max_flow) — can be slow if capacities are large!

Edmonds-Karp Algorithm

The Edmonds-Karp algorithm is Ford-Fulkerson with BFS to find the shortest augmenting path. This guarantees polynomial time.

function edmondsKarp(graph, source, sink, n): maxFlow = 0 parent = [-1] * n residual = copy(graph) while bfs(residual, source, sink, parent, n): // Find bottleneck bottleneck = infinity v = sink while v != source: u = parent[v] bottleneck = min(bottleneck, residual[u][v]) v = u // Augment flow v = sink while v != source: u = parent[v] residual[u][v] -= bottleneck residual[v][u] += bottleneck v = u maxFlow += bottleneck return maxFlow function bfs(residual, source, sink, parent, n): visited = [false] * n queue = [source] visited[source] = true parent[source] = -1 while queue not empty: u = queue.dequeue() for v = 0 to n-1: if not visited[v] and residual[u][v] > 0: visited[v] = true parent[v] = u if v == sink: return true queue.enqueue(v) return false

Time Complexity: O(V × E²) — polynomial and practical

Dinic's Algorithm

Dinic's algorithm (also Dinitz) is a more efficient max-flow algorithm that uses level graphs and blocking flows.

Key Concepts

  • Level Graph: Built using BFS, contains only edges going to the next level
  • Blocking Flow: A flow where every s-t path in the level graph is saturated

Algorithm Steps

  1. Build level graph using BFS from source
  2. Find blocking flow in level graph using DFS
  3. Add blocking flow to total flow
  4. Repeat until sink is unreachable from source

Time Complexity: O(V² × E) — O(E × √V) for unit capacity graphs

Applications

Bipartite Matching

Maximum matching in bipartite graphs reduces to max flow with unit capacities.

Network Routing

Finding maximum data throughput in computer networks.

Project Selection

Selecting projects with dependencies to maximize profit.

Image Segmentation

Min-cut algorithms for separating foreground from background.

Airline Scheduling

Assigning flights to minimize crew needed.

Sports Elimination

Determining if a team can still win the championship.

Bipartite Matching Example

To find maximum matching in a bipartite graph:

  1. Add source s connected to all vertices in set A with capacity 1
  2. Add sink t connected from all vertices in set B with capacity 1
  3. Original edges from A to B have capacity 1
  4. Max flow = Maximum matching size

Algorithm Comparison

Algorithm Time Complexity Best For
Ford-Fulkerson (DFS) O(E × max_flow) Small capacities
Edmonds-Karp (BFS) O(V × E²) General graphs
Dinic's O(V² × E) Dense graphs
Dinic's (unit cap) O(E × √V) Bipartite matching
Push-Relabel O(V² × E) or O(V³) Very dense graphs

Minimum Cut Finding

After running max flow, the minimum cut can be found by:

  1. Run BFS/DFS from source in the residual graph
  2. All reachable vertices are in set S
  3. All unreachable vertices are in set T
  4. Edges from S to T in original graph form the min cut

Min Cut = Max Flow

The total capacity of edges in the min cut equals the max flow value. These edges are "saturated" — they carry flow equal to their capacity.
