Understanding Recursion Stack and Backtracking Logic.

Recursion

Recursion is a method where the solution to a problem depends on solutions to smaller instances of the same problem. A recursive function calls itself.

Key Components

  • Base Case: The condition that stops the recursion (e.g., n == 0).
  • Recursive Case: The part where the function calls itself with a modified argument.
  • Call Stack: The memory structure that stores active function calls.

Example: Factorial

fact(n) = n * fact(n-1)

Calculates 5! by calling 4!, which calls 3!, etc., until 1! returns 1.

Backtracking

Backtracking is a refined brute force approach. It builds candidates for a solution recursively and abandons a candidate ("backtracks") as soon as it determines that the candidate cannot lead to a valid solution.

The 3 Keys

  • Choice: What option do we take at this step? (e.g., Place Queen at Col C).
  • Constraint: Is this choice valid? (e.g., Is position safe?).
  • Goal: Have we reached the solution? (e.g., Are N queens placed?).

N-Queens Complexity

Time Complexity O(N!)
Space Complexity O(N)
Exponential time makes it slow for N > 20.
