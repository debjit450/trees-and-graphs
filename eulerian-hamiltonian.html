Eulerian & Hamiltonian Paths

Next: Articulation & Bridges

Overview

This topic covers two classic graph problems that ask whether we can traverse a graph in a specific way:

Eulerian Path/Circuit

Visit every edge exactly once. Named after Leonhard Euler (1736).

Hamiltonian Path/Cycle

Visit every vertex exactly once. Named after William Hamilton (1857).

Historical Note

The Eulerian problem originated from the famous "Seven Bridges of Königsberg" puzzle. Euler proved it was impossible to walk through the city crossing each bridge exactly once, founding graph theory in the process!

Eulerian Paths and Circuits

Definitions

  • Eulerian Path: A path that visits every edge exactly once
  • Eulerian Circuit (Cycle): An Eulerian path that starts and ends at the same vertex
  • Eulerian Graph: A graph that has an Eulerian circuit
  • Semi-Eulerian Graph: A graph that has an Eulerian path but not a circuit

Conditions for Eulerian Paths/Circuits

Undirected Graphs

Eulerian Circuit Condition

An undirected connected graph has an Eulerian circuit if and only if every vertex has even degree.

Eulerian Path Condition

An undirected connected graph has an Eulerian path if and only if it has exactly 0 or 2 vertices of odd degree. If 2 vertices have odd degree, the path must start at one and end at the other.

Directed Graphs

Eulerian Circuit Condition

A directed graph has an Eulerian circuit if and only if:

  • The graph is strongly connected (ignoring edge directions for connectivity check)
  • Every vertex has in-degree = out-degree

Eulerian Path Condition

A directed graph has an Eulerian path if:

  • At most one vertex has out-degree - in-degree = 1 (start vertex)
  • At most one vertex has in-degree - out-degree = 1 (end vertex)
  • All other vertices have in-degree = out-degree

Hierholzer's Algorithm

Hierholzer's Algorithm efficiently finds an Eulerian path or circuit in O(E) time by building the path iteratively and inserting sub-circuits.

Algorithm Steps

  1. Start at any vertex (for circuit) or at a vertex with odd degree (for path)
  2. Follow edges, removing them as you go, until you return to start
  3. If there are remaining edges, find a vertex in current path with unused edges
  4. Build a sub-circuit from that vertex and splice it into the main path
  5. Repeat until all edges are used
function hierholzer(graph, start): stack = [start] path = [] while stack not empty: v = stack.top() if graph[v] has edges: // Take any edge from v u = graph[v].pop() // Remove edge v→u (for undirected, also remove u→v) removeEdge(graph, u, v) // For undirected graphs stack.push(u) else: // No more edges from v, add to path path.append(stack.pop()) return reverse(path) // For undirected graph: check condition first function findEulerianPath(graph): oddDegree = [] for v in vertices: if degree[v] % 2 == 1: oddDegree.append(v) if len(oddDegree) == 0: return hierholzer(graph, anyVertex) // Circuit else if len(oddDegree) == 2: return hierholzer(graph, oddDegree[0]) // Path else: return null // No Eulerian path exists

Time Complexity: O(E) — each edge processed exactly once

Space Complexity: O(E) for path storage

Hamiltonian Paths and Cycles

Definitions

  • Hamiltonian Path: A path that visits every vertex exactly once
  • Hamiltonian Cycle: A Hamiltonian path that returns to the starting vertex
  • Hamiltonian Graph: A graph that has a Hamiltonian cycle

Hamiltonian Problem Complexity

NP-Complete Problem

Unlike Eulerian paths (polynomial time), determining whether a Hamiltonian path or cycle exists is NP-complete. There is no known polynomial-time algorithm!

Why It's Hard

  • No simple degree-based conditions (unlike Eulerian)
  • Must potentially explore exponentially many paths
  • Related to the Traveling Salesman Problem (TSP)

Sufficient Conditions (Not Necessary)

These conditions guarantee a Hamiltonian cycle exists, but graphs not meeting them may still have one:

Theorem Condition
Dirac's Theorem (1952) If every vertex has degree ≥ n/2, graph has Hamiltonian cycle
Ore's Theorem (1960) If deg(u) + deg(v) ≥ n for all non-adjacent u,v, graph has Hamiltonian cycle
Complete Graph Kn Always has Hamiltonian cycle (for n ≥ 3)

Backtracking Algorithm for Hamiltonian

The brute-force approach uses backtracking to explore all possible paths:

function hamiltonianCycle(graph, n): path = [-1] * n path[0] = 0 // Start from vertex 0 if backtrack(graph, path, 1, n): return path return null // No Hamiltonian cycle function backtrack(graph, path, pos, n): // Base case: all vertices included if pos == n: // Check if cycle: edge from last to first return graph[path[pos-1]][path[0]] == 1 // Try each vertex as next in path for v = 1 to n-1: if isSafe(v, graph, path, pos): path[pos] = v if backtrack(graph, path, pos + 1, n): return true path[pos] = -1 // Backtrack return false function isSafe(v, graph, path, pos): // Check if v is adjacent to previous vertex if graph[path[pos-1]][v] == 0: return false // Check if v already in path for i = 0 to pos-1: if path[i] == v: return false return true

Time Complexity: O(n!) in worst case — exponential

Applications

Eulerian Applications

  • DNA Sequencing: Reconstructing DNA from fragments (de Bruijn graphs)
  • Circuit Design: Testing all connections on a circuit board
  • Snow Plow Routing: Covering all streets efficiently
  • Network Routing: Visiting all network links

Hamiltonian Applications

  • Traveling Salesman: Finding shortest tour visiting all cities
  • Genome Assembly: Ordering contigs in sequencing
  • Knight's Tour: Chess puzzle visiting all squares
  • Manufacturing: Optimizing machine tool paths
