B-Tree Properties

Definition

A B-tree of order t is a self-balancing tree where:

  • Every node has at most 2t-1 keys
  • Every non-root node has at least t-1 keys
  • Root has at least 1 key
  • All leaves are at the same level

Insertion

function insert(key): if root is full: // Split root, tree grows up newRoot = new Node() newRoot.children[0] = root splitChild(newRoot, 0) root = newRoot insertNonFull(root, key) function splitChild(parent, i): // Split child at index i // Move median up to parent // Create new sibling

Key insight: Always split full nodes proactively during descent.

B-Tree vs B+ Tree

Feature B-Tree B+ Tree
Data storage All nodes Leaves only
Leaf links None Linked list
Range queries Slower Fast
Internal nodes Data + keys Keys only
Disk usage Higher Lower

Applications

  • Database indexes (MySQL, PostgreSQL)
  • File systems (NTFS, ext4)
  • Directory structures
  • Key-value stores

Complexity

Search/Insert/Delete: O(log n)

Space: O(n)

All operations require O(logt n) disk accesses in practice.
