Dynamic Programming: Optimizing Recursion through Caching.

What is DP?

Dynamic Programming is a technique used to solve problems that have Overlapping Subproblems and Optimal Substructure.

Two Main Approaches

  • Memoization (Top-Down): Write the recursive solution, then add a cache/map to store results of function calls. If a state is visited again, return the cached value.
  • Tabulation (Bottom-Up): Solve the smallest subproblems first (base cases) and use their results to iteratively build up to the final answer.

When to use?

Whenever you see a recursive solution that solves the same subproblem multiple times (e.g., Fibonacci(5) calls Fibonacci(3) twice).

Classic Problem: 0/1 Knapsack

Given a set of items, each with a weight and a value, determine the number of each item to include in a collection so that the total weight is less than or equal to a given limit and the total value is as large as possible.

State Definition

dp[i][w] = Max value using subset of first i items with capacity w.

Recurrence Relation

If weight[i] > w:
dp[i][w] = dp[i-1][w] (Skip item)

Else:
dp[i][w] = max(dp[i-1][w], val[i] + dp[i-1][w-wt[i]])

Complexity Analysis

Time Complexity O(N * W)
Space Complexity O(N * W)
Can be space-optimized to O(W).

Implementation (Tabulation)

for (i = 1; i <= n; i++) { for (w = 0; w <= capacity; w++) { if (weights[i-1] <= w) { dp[i][w]=max( dp[i-1][w], // Exclude values[i-1] + dp[i-1][w - weights[i-1]] // Include ); } else { dp[i][w] = dp[i-1][w]; } } }
