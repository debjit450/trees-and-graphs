Big-O Notation

Definition

Big-O notation describes the upper bound of an algorithm's growth rate. We drop constants and lower-order terms.

  • O(1) - Array access, hash lookup
  • O(log n) - Binary search, balanced BST
  • O(n) - Linear search, single loop
  • O(n log n) - Merge sort, heap sort
  • O(n²) - Bubble sort, nested loops
  • O(2ⁿ) - Recursive fibonacci, subsets
  • O(n!) - Permutations, brute force TSP

Quick Rules

Identifying Complexity

  • Single loop → O(n)
  • Nested loops → O(n²), O(n³)...
  • Divide in half → O(log n)
  • Divide + merge → O(n log n)
  • Each element twice → still O(n)

Space Complexity

Auxiliary space = extra space beyond input

Total space = auxiliary + input

  • O(1) - In-place algorithms
  • O(log n) - Recursive call stack
  • O(n) - Linear data structures
  • O(n²) - 2D matrices
