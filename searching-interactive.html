Linear Search

The simplest search algorithm. It iterates through the collection one element at a time until the target is found or the end is reached.

Characteristics

  • Universal: Works on unsorted and sorted arrays, linked lists, etc.
  • Slow for Large Data: O(N) means checking 1 million items can take 1 million steps.

When to use?

When the list is small (N < 100) or when the data is unsorted and you only need to search once (sorting takes O(N log N) which is slower than one O(N) scan).

Binary Search

A highly efficient algorithm that works by repeatedly dividing the search interval in half. Requires a sorted array.

How it Works

  1. Compare target with the Middle element.
  2. If Equal, return index.
  3. If Target < Middle, discard right half. Search Left.
  4. If Target > Middle, discard left half. Search Right.

Complexity Analysis

Best Case O(1)
Average Case O(log N)
Worst Case O(log N)

Implementation (Iterative)

int binarySearch(int[] arr, int target) { int low = 0, high = arr.length - 1; while (low <= high) { int mid = low + (high - low) / 2; if (arr[mid] == target) return mid; if (arr[mid] < target) low=mid + 1; else high = mid - 1; } return -1; // Not Found }

Ternary Search

Similar to Binary Search, but divides the array into three parts instead of two. It maintains two mid-points (m1 and m2).

While theoretical complexity is O(log₃ N), it makes more comparisons per step than Binary Search, often making it slightly slower in practice for simple arrays. However, it is crucial for finding maxima/minima in unimodal functions.
